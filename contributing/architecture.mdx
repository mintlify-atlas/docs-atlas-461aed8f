Clash Verge Rev is split into two main parts: a **Rust backend** (in `src-tauri/`) that manages the Mihomo proxy core, system integration, and persistent config, and a **TypeScript/React frontend** (in `src/`) that renders the UI. Tauri bridges them via typed commands. Understanding this separation is key to contributing effectively.

## Frontend (`src/`)

The frontend is a standard React 19 + Vite application using Material UI for components and React Router for navigation. Pages map directly to the sidebar tabs:

| Page | File | Description |
|---|---|---|
| Home | `src/pages/home.tsx` | Traffic stats, proxy mode toggle, quick controls |
| Proxies | `src/pages/proxies.tsx` | Proxy groups and node selection |
| Profiles | `src/pages/profiles.tsx` | Profile management and enhancement |
| Rules | `src/pages/rules.tsx` | Active rules viewer |
| Connections | `src/pages/connections.tsx` | Live connection monitor |
| Logs | `src/pages/logs.tsx` | Mihomo core log stream |
| Settings | `src/pages/settings.tsx` | All app and core settings |

Communication with the backend uses `@tauri-apps/api`'s `invoke()` function, wrapped in typed service functions under `src/services/`.

## Backend (`src-tauri/src/`)

### `cmd/` вЂ” Tauri commands

Each file exposes `#[tauri::command]` functions that the frontend can call via `invoke()`. Commands are organized by domain:

| File | Responsibility |
|---|---|
| `profile.rs` | Get/create/update/delete profiles, activate profiles |
| `verge.rs` | Read and patch app settings (verge.yaml) |
| `clash.rs` | Read/patch Mihomo core config |
| `proxy.rs` | Get proxies, select proxy, test latency |
| `backup.rs` | Create/restore/list backups |
| `webdav.rs` | WebDAV backup upload/download |
| `network.rs` | Network diagnostics |
| `service.rs` | System service install/uninstall/start/stop |

### `config/` вЂ” Config structs

- **`IVerge`** (`verge.rs`) вЂ” App-level settings persisted to `verge.yaml`. Controls theme, proxy guard, TUN, hotkeys, ports, backup schedule, etc.
- **`IProfiles`** (`profiles.rs`) вЂ” The list of all profiles and which is active.
- **`PrfItem`** (`prfitem.rs`) вЂ” A single profile entry (type, URL, update interval, file content).
- **`ClashConfig`** (`clash.rs`) вЂ” Mihomo core config overlay (ports, mode, log level, etc.).

### `core/` вЂ” Runtime management

- **`CoreManager`** (`manager/`) вЂ” Starts, stops, and restarts the Mihomo binary sidecar.
- **`service.rs`** вЂ” The system helper service for privilege escalation (TUN mode).
- **`hotkey.rs`** вЂ” Registers and handles global keyboard shortcuts.
- **`tray/`** вЂ” System tray icon and menu management.
- **`updater.rs`** вЂ” Auto-update logic using `tauri-plugin-updater`.
- **`sysopt.rs`** вЂ” Sets and guards system proxy OS settings.

### `enhance/` вЂ” Profile enhancement pipeline

When a profile is activated, the enhancement pipeline runs:
1. Load the base profile (remote/local YAML)
2. Apply all **Merge** profiles in list order
3. Apply all **Script** profiles in list order
4. Write the resulting config to the runtime clash config file
5. Signal Mihomo to reload
